Make `this.s` writable on `ShapeHook`/`ShapeComponent`: assigning `this.s.foo = value` is now equivalent to `this.setState({foo: value})` and runs through the same re-render path. The proxy throws when assigning to an unregistered state key, so typos and missing `useState`/class-field declarations fail loudly instead of silently mutating `this.state`. Because `setState` already writes to `this.state` synchronously before scheduling React's update, a read of `this.s.foo` on the next line reflects the new value. `this.p` remains read-only.
